# 混合方案实施计划: SVG 底图 + 游戏坐标直接映射

> **For Hermes:** 使用 delegate_task 逐个执行

**Goal:** 把 `tarkov-map.html` 的底图从本地 JPG 手绘图换成 SVG 矢量图，利用 SVG viewBox 作为 CRS 坐标系，实现游戏坐标→地图像素的精确映射（误差 ~5%），缺失 SVG 的 4 张地图保留 JPG 回退。

**Architecture:**
```
SVG viewBox ──→ Leaflet CRS.Simple bounds
游戏 {x,z} ──→ rotation ──→ CRS 坐标 ──→ 图像像素
                                          ↓
                                Leaflet imageOverlay(SVG)
```

**Tech Stack:** Leaflet.js, SVG (矢量底图), 本地 JSON 数据

## Phase 2: 修改 Leaflet 底图为 SVG imageOverlay

### Task 2.1: 替换底图加载逻辑

**当前代码** (JPG):
```javascript
L.imageOverlay("maps/2d/Customs.jpg", [[0,0],[4025,7680]]).addTo(map);
```

**目标代码** (SVG):
```javascript
L.imageOverlay("maps/svg/Customs.svg", [[0,0],[535.17,1062.48]]).addTo(map);
```
注意：viewBox 的 `y, x` 映射到 CRS.Simple 的 `[lat, lng]` = `[y, x]`。

## Phase 3: 重构坐标映射逻辑

### Task 3.1: 更新 generate_positions.py

**关键改动：** 坐标映射从 JPG 像素映射改为 viewBox 直接映射。

**当前逻辑 (JPG):**
```python
img_y = h * (margin + (1-2*margin) * (0.5 - ry))
img_x = w * (margin + (1-2*margin) * (rx + 0.5))
```
(其中 w/h 是 JPG 像素)

**新逻辑 (SVG):**
```python
# 使用 viewBox 尺寸作为 CRS 边界
# 游戏坐标旋转后直接放入 viewBox 范围
margin = 0.05  # 更小的 margin
crx = 0.5 + rx  # 旋转后的 x (0-1 范围)
cry = 0.5 + ry  # 旋转后的 y (0-1 范围)
# viewBox 坐标系内：CRS 坐标 = norm * viewBoxDimension
crs_x = crx * viewBoxW
crs_y = cry * viewBoxH
# 任务标记的 position = [CRS_Y, CRS_X] (Leaflet CRS.Simple 格式)
position = [crs_y, crs_x]
```

**注意：** 任务标记的 `position` 现在是 CRS 坐标（Leaﬂet CRS.Simple `[y, x]`），不是 JPG 像素了！
